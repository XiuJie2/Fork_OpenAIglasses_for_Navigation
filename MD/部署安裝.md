## 系統需求

| 項目 | 最低要求 | 建議 |
|------|---------|------|
| 作業系統 | Windows 10 / Ubuntu 20.04 | Windows 11 / Ubuntu 22.04 |
| Python | **3.11（必須，嚴格限制）** | 3.11 |
| GPU | 無（CPU 模式，效能較差） | NVIDIA GPU + CUDA 11.8 |
| RAM | 8 GB | 16 GB 以上 |
| 硬碟空間 | 10 GB（模型 + 環境） | 20 GB |
| 網路 | 需要（下載模型與呼叫 API） | — |
| 硬體裝置 | 電腦模擬器（開發測試）| Seeed Studio XIAO ESP32S3 |

## 步驟一：安裝基礎工具

### Python 3.11

```bash
# Ubuntu
sudo apt update
sudo apt install python3.11 python3.11-venv

# Windows：至 https://www.python.org/downloads/release/python-3119/ 下載安裝
# 安裝時勾選「Add Python to PATH」
```

### uv（套件管理工具）

```bash
# Windows（PowerShell）
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

## 步驟三：設定環境變數

```bash
# 複製範本
cp .env.example .env   # Linux/macOS
copy .env.example .env  # Windows
```

用文字編輯器開啟 `.env`，**至少填入以下必填項目**：

```env
# ===== 必填：AI 服務 API Key =====
GROQ_API_KEY=gsk_你的Groq金鑰          # 語音辨識（Whisper）
GEMINI_API_KEY=AIza你的Gemini金鑰      # 語音對話 + TTS
DASHSCOPE_API_KEY=sk-你的阿里雲金鑰    # ASR 備援（可選）
OPENROUTER_API_KEY=sk-or-你的金鑰      # 場景描述備援（可選）

# ===== 模型路徑（預設值通常不需要改）=====
BLIND_PATH_MODEL=model/yolo-seg.pt
OBSTACLE_MODEL=model/yoloe-11l-seg.pt
TRAFFICLIGHT_MODEL=model/trafficlight.pt
SHOPPING_MODEL=model/shoppingbest5.pt
HAND_LANDMARKER_PATH=model/hand_landmarker.task
```

### API Key 取得網址

| 服務 | 網址 | 說明 |
|------|------|------|
| Groq | https://console.groq.com | 免費方案足夠使用（語音辨識） |
| Gemini | https://aistudio.google.com | 免費方案足夠使用（對話 + TTS） |
| DashScope | https://dashscope.aliyun.com | 阿里雲，ASR 備援用，可選 |
| OpenRouter | https://openrouter.ai | 場景描述備援用，可選 |

## 步驟四：安裝 Python 套件

```bash
# 建立虛擬環境並安裝所有套件（約 5~15 分鐘，視網速）
uv sync
```

### 若有 NVIDIA GPU（可選，提升 AI 推理速度）

```bash
# 安裝 CUDA 版 PyTorch（CUDA 11.8）
uv run pip install torch==2.0.1+cu118 torchvision==0.15.2+cu118 \
  --index-url https://download.pytorch.org/whl/cu118
```

確認 GPU 是否可用：

```bash
uv run python -c "import torch; print('GPU:', torch.cuda.is_available())"
```

## 步驟五：下載 AI 模型

```bash
# 自動下載所有模型到 model/ 目錄
uv run python download_models.py
```

若下載失敗，請手動至以下網址下載，並放入 `model/` 目錄：

```
https://www.modelscope.cn/models/archifancy/AIGlasses_for_navigation
```

模型目錄結構應如下：

```
model/
├── yolo-seg.pt           # 盲道分割偵測
├── yoloe-11l-seg.pt      # 開放詞彙障礙物偵測
├── shoppingbest5.pt      # 物品識別（購物場景）
├── trafficlight.pt       # 紅綠燈偵測
└── hand_landmarker.task  # Google MediaPipe 手部偵測
```

## 步驟六：啟動伺服器

```bash
uv run python app_main.py
```

啟動成功後會看到：

```
INFO:     Uvicorn running on http://0.0.0.0:8081 (Press CTRL+C to quit)
[AUTH] 資料庫初始化完成（預設帳號 admin / 1124）
[DISCOVERY] UDP 廣播已啟動（port 47777，每 2 秒）
```

瀏覽器開啟 `http://localhost:8081` 即可看到監控頁面。

### 使用電腦模擬器測試（無 ESP32 硬體）

需要開兩個終端機，**順序不可對調**：

**終端機 1 — 主伺服器**（先啟動）：
```bash
uv run python app_main.py
```

**終端機 2 — ESP32 模擬器**（後啟動）：
```bash
uv run python esp32_simulator.py
```

模擬器以電腦的攝影機、麥克風、喇叭取代 ESP32 硬體，功能與實機相同。

### 硬體連接（ESP32）

使用 Arduino IDE 開啟 `compile/compile.ino`，選擇板子為 **Seeed Studio XIAO ESP32S3**，在以下區段填入 Wi-Fi 與伺服器資訊後燒錄：

```cpp
const char* ssid = "你的WiFi名稱";
const char* password = "你的WiFi密碼";
const char* server_host = "192.168.x.x";  // Python 伺服器的 IP
const int server_port = 8081;
```

## 步驟七：連線 APP（Android）

### 區網連線（同一 Wi-Fi）

1. 查詢電腦 IP：
   ```bash
   # Windows
   ipconfig
   # Linux/macOS
   ip addr show
   ```
2. APP 設定頁 → **IP 模式** → 輸入 IP（例如 `192.168.1.100`），Port `8081`

### 公網連線（Cloudflare Tunnel）

```bash
# 安裝（一次性）
winget install Cloudflare.cloudflared   # Windows
brew install cloudflared                # macOS

# 啟動（每次使用）
cloudflared tunnel --url http://localhost:8081
```

複製輸出的 `https://xxxx.trycloudflare.com` 網址，貼到 APP 設定頁的 **URL 模式**。

## 常見問題

**Q：啟動報錯 `No module named 'xxx'`**

```bash
# 重新同步套件
uv sync
```

**Q：啟動時出現 `未設定 DASHSCOPE_API_KEY`**

確認 `.env` 檔案存在於專案根目錄，且相關 API Key 已正確填入。

**Q：GPU 記憶體不足（CUDA out of memory）**

在 `.env` 加入：

```env
AIGLASS_DEVICE=cpu      # 改用 CPU
AIGLASS_GPU_SLOTS=1     # 減少同時推理數
```

**Q：模型載入失敗（FileNotFoundError）**

確認 `model/` 目錄有所有 `.pt` 與 `.task` 檔案：

```bash
ls model/
# 或重新下載
uv run python download_models.py
```

**Q：Windows 上 PyAudio 安裝失敗**

```bash
uv run pip install pipwin
uv run pipwin install pyaudio
```

**Q：ESP32 無法連線**

確認 ESP32 韌體中的 Wi-Fi 名稱、密碼、伺服器 IP 是否正確，且 ESP32 與伺服器在同一網路下。

**Q：視覺查詢（幫我看）沒有回應**

確認 `OPENROUTER_API_KEY` 已設定。若未設定則會降級使用 Gemini（需要 `GEMINI_API_KEY`）。

**Q：想在沒有 ESP32 的情況下測試**

使用電腦模擬器：先啟動主伺服器，再執行 `uv run python esp32_simulator.py`。
