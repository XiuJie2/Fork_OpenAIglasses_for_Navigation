## 技術架構

```
Nginx (:80)
  ├── /api/, /admin/  →  Django 4.2（Gunicorn）
  ├── /media/         →  靜態媒體（GLB 模型、圖片）
  ├── /static/        →  Django 靜態檔案
  └── /               →  React 18（Vite）
```

| 層級 | 技術 |
|------|------|
| 前端 | React 18 + Vite + Tailwind CSS v3 + @react-three/fiber（3D 模型） |
| 後端 | Django 4.2 + DRF + SimpleJWT |
| 資料庫 | PostgreSQL 15 |
| 反向代理 | Nginx |
| 容器編排 | Docker Compose |

## 快速啟動

### 步驟一：設定環境變數

```bash
cd Website
cp .env.example .env
```

編輯 `.env`，**至少需要設定以下項目**：

```env
# PostgreSQL 資料庫
POSTGRES_DB=glasses_db
POSTGRES_USER=你的DB使用者名稱
POSTGRES_PASSWORD=你的DB密碼

# Django 安全金鑰（可用下列指令產生）
# python -c "import secrets; print(secrets.token_hex(32))"
SECRET_KEY=your-secret-key-here

# 後台超級管理員帳號（首次啟動自動建立）
DJANGO_SUPERUSER_USERNAME=admin
DJANGO_SUPERUSER_EMAIL=admin@example.com
DJANGO_SUPERUSER_PASSWORD=your-password-here
```

### 步驟二：啟動所有服務

```bash
docker compose up --build
```

首次啟動會自動執行：
1. 等待 PostgreSQL 就緒
2. 執行資料庫 migrate
3. 建立超級管理員帳號（依 `.env` 設定）
4. 載入初始資料（產品、功能特色、規格、團隊成員、頁面文字）

## 後台管理說明

進入 `http://localhost/admin/login`，使用 `.env` 中設定的帳號登入。

### 角色說明

| 角色 | 說明 |
|------|------|
| 超級管理員（superadmin） | 可存取所有功能，包含帳號管理與所有內容 |
| 一般管理員（admin） | 僅能存取超級管理員授權的功能區塊 |

### 各功能區塊操作說明

**流量追蹤**：查看每日/每週訪客趨勢，了解各頁面流量熱度。

**頁面內容管理**：修改網站各頁面的文字內容，無需改程式碼即可即時更新（詳見下方模板系統說明）。

**商品管理**：
- 修改商品基本資訊（名稱、說明、價格）
- 管理功能特色列表
- 修改規格表項目

**訂單管理**：查看訪客送出的購買表單，包含聯絡資訊與數量。

**成員管理**：新增或修改團隊頁面的成員卡片，可上傳大頭照。

**帳號管理**（僅超級管理員）：新增管理員帳號、停用帳號、調整帳號角色。

**操作日誌**：所有後台操作均會記錄，可追蹤誰在何時做了什麼修改。

**APP 裝置管理**：透過此介面管理連接 FastAPI 主伺服器的使用者帳號與緊急連絡人資料。

## 頁面內容模板系統

網站所有頁面的文字內容均可透過後台「頁面內容管理」即時修改，**修改後立即生效，無需重新建置**。

可管理的頁面內容：

| 頁面 / 區塊 | 可修改的項目 |
|-------------|-------------|
| 全站設定 | 品牌名稱、導覽列連結文字、頁尾版權資訊 |
| 首頁 | Hero 標題、說明文字、統計數字（例如「支援 N 種語言」）、CTA 按鈕文字 |
| 產品介紹頁 | 頁面主標題、各標籤頁名稱 |
| APP 下載頁 | 下載卡片標題說明、功能特色清單、安裝步驟文字 |
| 購買頁 | 表單欄位標籤、送出按鈕文字、購買完成頁面文字 |
| 團隊頁 | 頁面標題、各區塊說明文字 |

## 日常操作指令

```bash
# 啟動（背景執行）
docker compose up -d

# 停止
docker compose down

# 查看即時日誌
docker compose logs -f

# 只看後端日誌
docker compose logs -f backend

# 只看前端日誌
docker compose logs -f frontend

# 重新建置（修改 Dockerfile 或 requirements 後執行）
docker compose up --build

# 進入 Django shell
docker compose exec backend python manage.py shell

# 手動建立 migration
docker compose exec backend python manage.py makemigrations accounts products orders team content analytics

# 手動執行 migrate
docker compose exec backend python manage.py migrate

# 重設超級管理員密碼
docker compose exec backend python manage.py shell -c "
from accounts.models import CustomUser
u = CustomUser.objects.get(username='admin')
u.set_password('新密碼')
u.save()
print('密碼已更新')
"
```

## 專案結構

```
Website/
├── aiglass.glb              # AI 眼鏡 3D 模型（唯讀掛載至容器）
├── docker-compose.yml       # 服務編排
├── .env                     # 環境變數（不提交 Git）
├── .env.example             # 環境變數範本
├── nginx/
│   └── nginx.conf           # Nginx 路由設定
├── backend/                 # Django 後端
│   ├── accounts/            # 自訂使用者模型（superadmin/admin）
│   ├── products/            # 商品、功能特色、規格
│   ├── orders/              # 訂單管理
│   ├── team/                # 團隊成員
│   ├── content/             # 頁面內容 Singleton 模型（模板系統）
│   ├── analytics/           # 流量追蹤與操作日誌
│   └── entrypoint.sh        # 啟動腳本（自動 migrate、建帳號、載入初始資料）
└── frontend/                # React 前端
    └── src/
        ├── pages/           # Home / Product / Download / Purchase / Team
        ├── components/      # Navbar / Footer / ModelViewer（3D）/ ScrollToTop
        └── admin/           # 後台管理介面（Login / Dashboard / CRUD 區塊）
```

## 注意事項

- `.env` 已加入 `.gitignore`，請勿手動提交至版本庫
- `aiglass.glb` 為 3D 模型二進位檔，如需在本機預覽可使用 [glTF Viewer](https://gltf-viewer.donmccurdy.com/)
- 後台帳號密碼若遺忘，可透過上方 shell 指令重設
- 修改頁面文字後立即生效（Singleton 模型機制）；新增/刪除商品功能/規格則透過後台表單操作
- 展示網站與 AI 眼鏡主伺服器（`app_main.py`）為**獨立服務**，可分開部署在不同機器上
