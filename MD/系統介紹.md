## 系統簡介

AI 智慧眼鏡系統是一套**視障輔助導航平台**，整合即時電腦視覺、語音辨識與多模態 AI，協助視障使用者進行：

- 盲道沿線導航
- 斑馬線過馬路輔助（含紅綠燈偵測）
- 周遭物品尋找
- 場景描述與語音對話

系統核心為 Python FastAPI 伺服器，可搭配三種前端裝置使用（ESP32 眼鏡硬體、Android APP、電腦模擬器）。

## 可用裝置

| 裝置 | 說明 | 適用情境 |
|------|------|---------|
| **ESP32 硬體眼鏡** | Seeed Studio XIAO ESP32S3，穿戴式裝置 | 正式部署、實地使用 |
| **Android APP** | Flutter App，用手機取代硬體眼鏡 | 日常使用、無硬體時 |
| **電腦模擬器** | `esp32_simulator.py`，以電腦攝影機與麥克風模擬 ESP32 | 開發測試、不需任何硬體 |

三種裝置均透過 WebSocket 連接同一個 FastAPI 伺服器，功能完全相同。

## 五大核心功能

### 1. 盲道導航

沿著地面盲道（導盲磚）行走，系統自動辨識盲道位置並語音引導方向。

系統會持續語音播報：
- **直行**：「請繼續直行。」
- **偏移修正**：「請向左平移。」/ 「請向右平移。」
- **轉彎提示**：「請向左轉。」/ 「請向右轉。」
- **障礙物警告**：「前方有障礙物，注意避讓。」

技術實現：YOLO 盲道分割 + Lucas-Kanade 光流計算位移方向。

### 2. 過馬路輔助

偵測斑馬線幾何位置與紅綠燈狀態，引導使用者安全過馬路。

完整流程：
```
尋找斑馬線 → 對齊方向 → 等待綠燈 → 語音提示通過 → 尋找下一段盲道
```

### 3. 紅綠燈偵測

單獨啟動紅綠燈偵測模式，持續識別燈號並語音回報，不進行完整過馬路流程。

### 4. 物品尋找

使用 YOLO 開放詞彙偵測，協助找到使用者指定的物品。系統透過 AI 將中文物品名稱轉換為英文標籤，再送入 YOLO 進行偵測。

範例：說「找一下水杯」→ 系統辨識畫面中的杯子並提示位置。

### 5. 語音對話與場景描述

不符合導航指令的語音自動進入 AI 對話模式，可結合攝影機畫面進行問答與場景描述。

## 系統狀態機

系統任何時刻只會處於以下其中一個狀態（各功能互斥，切換時自動停止前一個模式）：

| 狀態 | 說明 |
|------|------|
| `IDLE` | 待機，等待語音喚醒 |
| `CHAT` | 自由對話模式（Gemini / Qwen-Omni） |
| `BLINDPATH_NAV` | 盲道導航中 |
| `CROSSING` | 過馬路流程中 |
| `ITEM_SEARCH` | 物品尋找中 |
| `TRAFFIC_LIGHT_DETECTION` | 紅綠燈偵測中 |

`BLINDPATH_NAV` 子狀態流程：
```
ONBOARDING（旋轉/平移校正）→ NAVIGATING → MANEUVERING_TURN / AVOIDING_OBSTACLE
```

`CROSSING` 子狀態流程：
```
SEEKING_CROSSWALK → WAIT_TRAFFIC_LIGHT → CROSSING → SEEKING_NEXT_BLINDPATH
```

## AI 服務分工

系統整合多個 AI 服務，依功能自動選擇最合適的模型：

| 功能 | AI 服務 | 所需 API Key |
|------|---------|-------------|
| 語音辨識（麥克風 → 文字） | Groq — Whisper Large v3 Turbo | `GROQ_API_KEY` |
| 物品名稱翻譯（中文 → 英文 YOLO 標籤） | Groq — Llama 3.1 8B Instant | `GROQ_API_KEY` |
| 場景描述（優先） | OpenRouter — Nemotron Nano 12B VLM | `OPENROUTER_API_KEY` |
| 場景描述（備援）/ 語音對話 | Gemini 2.5 Flash | `GEMINI_API_KEY` |
| TTS 語音合成 | Gemini TTS（Flash Preview） | `GEMINI_API_KEY` |
| 盲道分割偵測 | 本地 YOLO（yolo-seg.pt） | 不需 API |
| 障礙物偵測 | 本地 YOLO（yoloe-11l-seg.pt） | 不需 API |
| 紅綠燈偵測 | 本地 YOLO（trafficlight.pt） | 不需 API |

### 語音處理流程

```
使用者說話
    │
    ▼
Groq Whisper ASR（語音 → 文字）
    │
    ▼
關鍵字比對
    ├── 導航指令 ────────────────→ NavigationMaster 狀態機
    ├── 視覺查詢關鍵字 ──────────→ Nemotron VLM（失敗則降級 Gemini）
    ├── 找物品（找一下...） ─────→ Groq 提取標籤 → YOLO 偵測
    └── 其他自由對話 ────────────→ Gemini（含攝影機畫面）
```

## 監控頁面說明

主監控頁面（`http://IP:8081/`）提供：

- **即時攝影機畫面**：含 YOLO 偵測視覺化框線
- **語音辨識即時文字**：ASR 辨識到的文字即時顯示
- **系統狀態**：目前所處的狀態機狀態
- **AI 回覆訊息**：最新一次 AI 回應內容
- **IMU 3D 視覺化**：ESP32 姿態感測資料（若有連接 IMU）
- **聲紋管理入口**：右上角連結跳轉至聲紋 Dashboard

## 聲紋管理 Dashboard

聲紋管理頁面（`http://IP:8081/speaker`）提供：

| 功能 | 說明 |
|------|------|
| 聲紋錄製 | 按下「開始錄製聲紋」，對麥克風說話 10 秒 |
| 功能開關 | 說話人驗證 / 持續監測 / 旁路喚醒詞 各自的 Toggle |
| 即時儀表 | 音量 RMS 條形圖、相似度量表（含門檻白線） |
| 驗證測試 | 開始 3 秒單次驗證測試，即時顯示相似度分數 |
| 監測歷史 | 每次持續監測事件自動新增到歷史表格 |
| Debug 日誌 | 即時顯示所有 SSE 推送事件與時間戳 |

啟用聲紋驗證後，待機模式（等待喚醒詞「哈囉 曼波」）只有與錄製聲紋相似度超過門檻的聲音，才會送往語音辨識，可有效過濾路人或環境噪音誤觸發。

## 進階設定參數（.env）

以下為常用的 `.env` 覆寫參數，詳細完整列表請參考系統管理文件：

### 語音 / TTS

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `ENABLE_TTS` | `true` | 是否啟用語音播報 |
| `TTS_INTERVAL_SEC` | `1.0` | 相同語音最短間隔（秒） |
| `INTERRUPT_KEYWORDS` | `停下,別說了,停止` | 打斷熱詞（逗號分隔） |

### 麥克風靈敏度

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `ASR_PCM_GAIN` | `8.0` | ESP32 麥克風增益倍數（建議 5~10） |
| `ASR_STANDBY_RMS_THRESH` | `150.0` | 待機模式最低音量門檻（越低越靈敏） |

### 說話人聲紋驗證

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `SPEAKER_VERIFY_ENABLED` | `false` | 是否啟用說話人驗證 |
| `SPEAKER_THRESHOLD` | `0.82` | 相似度門檻（0~1，越高越嚴格） |

### GPU / 效能

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `AIGLASS_DEVICE` | `cuda:0` | 推理裝置（`cuda:0` 或 `cpu`） |
| `CONF_THRESHOLD` | `0.20` | YOLO 全域置信度門檻 |
